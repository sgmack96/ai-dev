## Theory (SE-Level)

R2 is Cloudflare's S3-compatible object storage with zero egress fees. That last part — zero egress fees — is the entire commercial pitch, and it needs no elaboration for any developer who has received an AWS billing surprise.

**The problem R2 solves:** AWS S3 charges $0.09 per GB for data transferred out of S3. If you're serving a 1 TB dataset to external users, that's $90 per TB in egress alone, every time. For video hosting, CDN origin, machine learning datasets, backups, or any high-read storage workload, this compounds fast.

**R2's pricing:**
- Storage: $0.015/GB/month
- Egress: **$0.00** (zero, none, free)
- Class A operations (write): $4.50/million requests
- Class B operations (read): $0.36/million requests
- Free tier: 10 GB storage, 1M Class A, 10M Class B per month

**S3 compatibility:** R2 exposes the same API as AWS S3. Any application already using an S3 SDK can switch to R2 by changing the endpoint URL and credentials — no code rewrite required. This is the "zero friction migration" pitch.

**Where R2 sits in the Cloudflare stack:** R2 integrates natively with Workers (via bindings), Pages (for static asset storage), Images (image storage layer), and Stream (video storage). It is also available as a standalone S3-compatible storage service for any application.

## Deep Dive (Architect-Level)

### Storage Architecture

R2 stores objects in Cloudflare's global infrastructure. Unlike S3's region model (us-east-1, eu-west-1), R2 operates on a "jurisdiction" concept with automatic data placement:

- **Default (automatic):** Cloudflare selects the storage location for performance; data may reside in any Cloudflare facility
- **EU jurisdiction:** Data stored within EU data centers (for GDPR compliance)
- **FedRAMP/Government:** Specific compliance configurations available

Objects are stored with redundancy across multiple physical locations within the jurisdiction.

### Worker Bindings

When a Worker accesses R2 via a binding, the read path is:
1. Worker receives request
2. Worker calls `env.MY_BUCKET.get("object-key")`
3. R2 returns the object from storage
4. Worker streams response to client

Because Workers run at the edge and R2 is globally distributed, read latency for cached/popular objects is very low. For large objects, streaming is used to avoid loading the entire object into Worker memory (limited to 128 MB).

### Object Model

| Concept | Details |
|---|---|
| **Key** | Object identifier, up to 1024 bytes, "/" is not a special character (use as pseudo-directory prefix) |
| **Value** | Binary data, up to 5 TB per object |
| **Metadata** | Up to 2 KB of custom key-value pairs |
| **HTTP headers** | Content-Type, Cache-Control, Content-Disposition, ETag |
| **Versioning** | Not available in R2 (unlike S3 versioning) |
| **Object tagging** | Available as metadata |

### Public Access Modes

| Mode | How | Use Case |
|---|---|---|
| **r2.dev subdomain** | Enable in dashboard, get `pub-{hash}.r2.dev` URL | Testing, quick public access |
| **Custom domain** | Add CNAME from your domain to R2 bucket | Production public CDN |
| **Presigned URL** | Generate time-limited signed URL in Worker | Temporary user download/upload |
| **Private (default)** | No public access, only via Worker binding or S3 API | Internal data, user files |

### Event Notifications

R2 can trigger notifications when objects are created or deleted:

```toml
# wrangler.toml — event notification to a Queue
[[r2_buckets]]
binding = "MY_BUCKET"
bucket_name = "macksportreport-uploads"

[[queues.consumers]]
queue = "r2-events"
```

This enables use cases like: image uploaded → trigger Workers AI for processing → store result → notify via Queues.

### Multipart Uploads

For objects over 100 MB, R2 supports S3-compatible multipart upload. This allows:
- Parallel upload of chunks for faster throughput
- Resume interrupted uploads
- Required for objects over 5 GB

### CORS Configuration

R2 supports CORS for browser-based direct uploads (presigned PUT URLs):

```json
[
  {
    "AllowedOrigins": ["https://macksportreport.com"],
    "AllowedMethods": ["GET", "PUT"],
    "AllowedHeaders": ["Content-Type"],
    "MaxAgeSeconds": 3600
  }
]
```

1. Click on your bucket
2. Click "Upload"
3. Drag and drop a file
4. Observe: key, size, type, ETag

**Step 3: Configure Public Access**
1. Settings → Public Access
2. Enable "Allow Access" with r2.dev subdomain
3. Copy the r2.dev URL
4. Test: `curl https://pub-{hash}.r2.dev/your-object-key`

**Step 4: Add a Custom Domain**
1. Settings → Custom Domains
2. Click "Connect Domain"
3. Enter: `files.macksportreport.com`
4. Cloudflare adds the CNAME automatically (since macksportreport.com is on CF)
5. Test: `curl https://files.macksportreport.com/your-object-key`

**Step 5: Generate API Tokens for S3 Access**
1. Manage R2 API Tokens → Create API Token
2. Select permissions: Object Read & Write (or Object Read Only)
3. Note the Access Key ID and Secret Access Key (shown only once)

**Step 6: View Analytics**
1. Dashboard → Analytics tab
2. See: storage used, requests, data transferred (always $0 egress)

### Lab 1: Create and Interact with a Bucket via Wrangler
```bash
# Create bucket
wrangler r2 bucket create macksportreport-test

# Upload a file
echo '{"message": "Hello from R2"}' > test.json
wrangler r2 object put macksportreport-test/hello.json --file test.json

# Download the object
wrangler r2 object get macksportreport-test/hello.json --file downloaded.json
cat downloaded.json

# List objects
wrangler r2 object list macksportreport-test

# Delete object
wrangler r2 object delete macksportreport-test/hello.json

# Delete bucket (must be empty)
wrangler r2 bucket delete macksportreport-test
```

### Lab 2: Use R2 via S3-Compatible API
```bash
# Install AWS CLI
pip install awscli  # or brew install awscli

# Configure for R2
aws configure set aws_access_key_id $R2_ACCESS_KEY_ID
aws configure set aws_secret_access_key $R2_SECRET_ACCESS_KEY
aws configure set default.region auto

# R2 endpoint
R2_ENDPOINT="https://${CF_ACCOUNT_ID}.r2.cloudflarestorage.com"

# List buckets
aws s3 ls --endpoint-url $R2_ENDPOINT

# Create bucket
aws s3 mb s3://macksportreport-s3-test --endpoint-url $R2_ENDPOINT

# Upload file
echo "Test content" > s3-test.txt
aws s3 cp s3-test.txt s3://macksportreport-s3-test/s3-test.txt --endpoint-url $R2_ENDPOINT

# List objects
aws s3 ls s3://macksportreport-s3-test --endpoint-url $R2_ENDPOINT

# Download
aws s3 cp s3://macksportreport-s3-test/s3-test.txt downloaded-s3.txt --endpoint-url $R2_ENDPOINT

# Sync a directory
aws s3 sync ./local-folder s3://macksportreport-s3-test/backup/ --endpoint-url $R2_ENDPOINT
```

### Lab 3: Worker with R2 Binding
```typescript
// src/index.ts
interface Env {
  UPLOADS: R2Bucket;
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const url = new URL(request.url);
    const key = url.pathname.slice(1); // remove leading /

    if (request.method === 'PUT') {
      // Upload object
      await env.UPLOADS.put(key, request.body, {
        httpMetadata: {
          contentType: request.headers.get('Content-Type') || 'application/octet-stream',
        },
        customMetadata: {
          uploadedAt: new Date().toISOString(),
          uploadedBy: request.headers.get('X-User-ID') || 'anonymous',
        },
      });
      return new Response(`Uploaded: ${key}`, { status: 201 });
    }

    if (request.method === 'GET') {
      const object = await env.UPLOADS.get(key);
      if (!object) {
        return new Response('Not Found', { status: 404 });
      }
      const headers = new Headers();
      object.writeHttpMetadata(headers);
      headers.set('ETag', object.httpEtag);
      return new Response(object.body, { headers });
    }

    if (request.method === 'DELETE') {
      await env.UPLOADS.delete(key);
      return new Response('Deleted', { status: 200 });
    }

    if (request.method === 'GET' && key === '') {
      // List all objects
      const list = await env.UPLOADS.list();
      return Response.json(list.objects.map(obj => ({
        key: obj.key,
        size: obj.size,
        uploaded: obj.uploaded,
      })));
    }

    return new Response('Method Not Allowed', { status: 405 });
  }
} satisfies ExportedHandler<Env>;
```

### Lab 4: Presigned URL Generation
```typescript
// Generate presigned URL for client-side upload
import { AwsClient } from 'aws4fetch';

interface Env {
  R2_ACCESS_KEY_ID: string;
  R2_SECRET_ACCESS_KEY: string;
  CF_ACCOUNT_ID: string;
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const r2 = new AwsClient({
      accessKeyId: env.R2_ACCESS_KEY_ID,
      secretAccessKey: env.R2_SECRET_ACCESS_KEY,
      service: 's3',
      region: 'auto',
    });

    const filename = new URL(request.url).searchParams.get('filename') || 'upload.bin';
    const uploadUrl = `https://${env.CF_ACCOUNT_ID}.r2.cloudflarestorage.com/macksportreport-uploads/${filename}`;

    // Create presigned URL valid for 1 hour
    const signed = await r2.sign(
      new Request(uploadUrl, { method: 'PUT' }),
      { aws: { signQuery: true } }
    );

    return Response.json({
      uploadUrl: signed.url,
      expiresIn: 3600,
      filename,
    });
  }
} satisfies ExportedHandler<Env>;
```

## Competitive Context

| Feature | Cloudflare R2 | AWS S3 | Google Cloud Storage | Azure Blob Storage | Backblaze B2 |
|---|---|---|---|---|---|
| **Egress cost** | **$0** | $0.09/GB | $0.12/GB | $0.087/GB | $0.01/GB |
| **Storage cost** | $0.015/GB | $0.023/GB | $0.020/GB | $0.018/GB | $0.006/GB |
| **S3 API compatibility** | Yes (full) | Native | Yes (via interop) | Partial | Yes |
| **Workers integration** | Native binding | Lambda + S3 (HTTP) | Cloud Functions + GCS | Azure Functions + Blob | Custom integration |
| **Global redundancy** | Automatic | Manual (multi-region) | Regional/dual-region | LRS/GRS/RA-GRS | US-only |
| **Versioning** | No (2024) | Yes | Yes | Yes | Yes |
| **Object lifecycle rules** | Limited | Full | Full | Full | No |
| **Event notifications** | Queues/Workers binding | SNS/SQS | Pub/Sub | Event Grid | Webhooks |
| **Custom domain CDN** | Native (CF CDN) | CloudFront (separate) | Cloud CDN (separate) | Azure CDN (separate) | Cloudflare CDN |
| **Free tier** | 10 GB / 1M Class A | 5 GB / 20K GET | 5 GB / 5K Class A | 5 GB / various | 10 GB |
| **EU data residency** | Yes (jurisdiction) | Yes (region selection) | Yes | Yes | No |

**Key positioning:** R2's zero egress is unmatched. For any read-heavy workload (CDN origin, ML datasets, media files, backups accessed regularly), R2 wins on price vs all three hyperscalers. The S3 compatibility means no vendor lock-in narrative works against it.
