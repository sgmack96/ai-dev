## Gap Fillers

Three small additions to existing sprint days that close the remaining gaps without changing the sprint's trajectory:

### Gap 1: Fine-tuning / Model Customization (add to Day 17 or 18)
**Relevance:** All 4 roles mention model customization. Anthropic asks about "fine-tuning or custom model usage."

**What to do:**
- Read Workers AI fine-tuning docs (or LoRA adapter patterns)
- Create a short note: `WEEK-03-production/evaluation/fine-tuning-notes.md`
- Document: when to fine-tune vs RAG vs prompt engineering. Cost/tradeoff analysis
- **Time:** 1 hour

### Gap 2: SE Discovery-to-Deployment Playbook (add to Day 22-23)
**Relevance:** Every role expects you to own the customer arc from discovery → POC → production. Currently we have a demo script but not the full playbook.

**What to do:**
- Create `career-alignment/se-playbook.md`
- Document: Discovery questions → Technical validation → POC scope → Deployment checklist → Production handoff
- Map each phase to a sprint artifact (e.g., "Discovery → Business Case Template", "POC → Golf Agent Demo")
- **Time:** 1.5 hours

### Gap 3: Account Health / Adoption Scorecard (add to Day 20)
**Relevance:** OpenAI Success Engineer role specifically requires "account health, adoption velocity, and ongoing technical success."

**What to do:**
- Add to the Day 20 enterprise readiness audit: a "Customer Health Scorecard" section
- Framework: Technical health (latency, error rate, uptime) + Adoption health (use cases, users, expansion) + Relationship health (NPS, stakeholder engagement)
- **Time:** 30 minutes
